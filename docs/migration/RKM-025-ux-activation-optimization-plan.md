## 0) 공통 목표 / 측정 정의

핵심 목표(Activation):
- 첫 세션 내 핵심 행동 1회 이상 달성 비율을 올린다.
- 핵심 행동: `vote`, `rsvp_join`, `post_create_success`

핵심 지표 공식:
- `activation_rate = first_session_users_with_core_action / first_session_users`
- `post_completion_rate = post_create_success / post_create_start`
- `feed_action_rate = users_with_vote_or_rsvp / feed_view_users`

공통 원칙:
- 신규 UX는 기능 플래그로 단계적 롤아웃
- 상태 변경은 카드 인라인 피드백 우선, 토스트는 보조
- 이벤트는 모두 `user_id`, `role`, `post_id(optional)`, `surface` 포함

## 1) 첫 방문 온보딩 플랜

### 1-1. 사용자 유형
- `완전 신규`: 가입 후 24시간 이내, 핵심 행동 0회
- `재방문 미참여`: 최근 14일 접속 이력 있음, 핵심 행동 0회
- `활동 사용자`: 최근 14일 핵심 행동 1회 이상

### 1-2. 3단계 온보딩 시나리오
1. 피드 탐색
2. 투표
3. RSVP 또는 글 작성

### 1-3. 마이크로카피 (한 줄 + 즉시 행동 버튼)
| 단계 | 안내 문구 | 버튼 |
| --- | --- | --- |
| 1 | 지금 인기 제안을 10초만 훑어보세요. | 피드 보기 |
| 2 | 마음에 드는 제안에 투표해 우선순위를 올리세요. | 투표하러 가기 |
| 3 | 참여 의사를 남기거나 직접 여행을 제안해보세요. | 참여하기 |

### 1-4. 노출/종료/재노출 규칙
- 첫 로그인 시 자동 노출
- 핵심 행동 1회 달성 시 온보딩 자동 종료
- `스킵`/`닫기` 선택 시 세션 종료 후 미노출
- 단, 7일 내 핵심 행동이 없으면 1회 재노출
- 완료 후에는 자동 재노출 없음, `다시 보기` 메뉴에서 수동 접근

### 1-5. 이벤트
- `onboarding_viewed`
- `onboarding_step_clicked` (`step=1|2|3`)
- `onboarding_skipped`
- `onboarding_completed`

### 1-6. 1주 모니터링 기준
- 단계별 이탈률 상위 구간(진입 대비 클릭률 기준)부터 카피/순서 개선

## 2) 글 작성 폼 단계화 플랜

### 2-1. 2단계 구조
- 1단계(필수): `장소`, `날짜`, `인원`
- 2단계(선택): `비용`, `준비물`, `상세`

### 2-2. 1단계 완료 조건(유효성)
- 장소: 2~60자
- 날짜: 현재 시점 이후
- 인원: 1~200 정수
- 위 3개가 모두 유효해야 2단계 진입 허용

### 2-3. 2단계 UX 규칙
- 기본 접힘 상태
- `건너뛰기` 버튼 제공
- 단계 표시: `1/2`, `2/2`

### 2-4. 중단 복원(임시저장)
- 저장 위치: `localStorage`
- 저장 시점: 필드 blur 또는 5초 idle
- 재접속 시 24시간 이내 draft 자동 복원
- 제출 성공 또는 사용자가 폐기 선택 시 draft 삭제

### 2-5. 에러 표시
- 단계 내 인라인 즉시 노출(필드 하단)
- 상단 요약 에러 배너는 미사용

### 2-6. 이벤트
- `post_create_start`
- `post_step1_valid`
- `post_step2_skipped`
- `post_create_success`
- `post_create_fail` (`reason`)

## 3) 카드 정보 우선순위 재정렬 플랜

### 3-1. 핵심 4정보
- 장소
- 날짜
- 마감
- 잔여 인원

### 3-2. 정보 아키텍처
- 1행(핵심): 장소 | 날짜 | 마감 | 잔여 인원
- 2행(보조): 작성자/카테고리/상태
- 보조 상세(준비물/비용/메모): `상세 보기` 접힘 영역

### 3-3. 상태 배지 규칙
- `확정`: 운영/작성자 확정 처리 완료
- `제안`: 기본 상태
- `마감임박`: 마감까지 24시간 이하
- `마감`: 마감 시각 경과 또는 잔여 인원 0

### 3-4. 스캔 효율 규칙
- 카드 높이 상한 고정(모바일/데스크톱 각각 토큰화)
- 리스트에서 핵심 텍스트 대비를 보조 정보보다 1단계 이상 강조

### 3-5. 측정
- 스크롤 1회당 사용자 시야 내 카드 수
- 상세 펼침률
- 카드 기반 행동률(`vote`, `rsvp`)

## 4) CTA 문구 명확화 플랜

상태 정의표:

| 상태 | 버튼 문구 | 보조 텍스트 |
| --- | --- | --- |
| 기본(참여 가능) | 참여하기 | 자리 여유 있음 |
| 활성(이미 참여) | 참여 취소 | 참여 중 |
| 대기열 가능 | 대기열 참여 | 현재 대기열 N명 |
| 대기열 참여 중 | 대기열 취소 | 내 순번 N번 |
| 마감 | 마감됨 | 참여 불가 |

운영 규칙:
- 유사 행동 용어 통일: `투표`, `참여`, `댓글`
- 부정형 문구보다 행동형 문구 우선
- 관리자와 일반 사용자 문구 톤 분리
- 클릭 결과 메시지 템플릿 통일(성공/실패)

검증 지표:
- 오해 클릭률(취소 후 즉시 재클릭, 동일 버튼 반복 클릭)

## 5) 필터 사용성 강화 플랜

### 5-1. 우선순위
- 탐색 시나리오 기준 1순위: 카테고리, 2순위: 일정/마감, 3순위: 정렬

### 5-2. 인터랙션 규칙
- 필터바 고정 기준 위치 정의(헤더 하단)
- 적용 결과 개수 표기: 예) `여행 12건`
- 활성 필터 칩 강조 + `초기화` 고정 노출
- 검색어+필터 충돌 시 최근 입력 우선, 무효 조합은 즉시 해제
- 모바일은 2행 또는 바텀시트로 제공

### 5-3. 이벤트/지표
- `filter_applied`, `filter_cleared`
- 필터 사용 후 클릭 전환률(`post_open`/`vote`/`rsvp`)

## 6) 개인화 기본 피드 플랜

### 6-1. 신호 정의

- 노출 사유 문구 제공: 예) `내가 투표한 제안`

### 6-5. 지표

- 체류시간

## 7) 마감/대기열 피드백 강화 플랜

### 7-1. RSVP 상태 전이

### 7-2. 피드백 정책

- 실패 시 원상복구 + 인라인 롤백 메시지

- 마감임박 기준: 24시간, 전용 강조 색상 사용
- 토스트 의존도 최소화, 카드 인라인 피드백 기본

### 7-3. 검증
- 상태 이해도 사용자 테스트
- 상태 라벨 오해율 추적

## 8) 빈 상태 UX 개선 플랜

### 8-1. 빈 상태 유형
- 데이터 없음
- 검색 결과 없음
- 필터 과다

### 8-2. 유형별 템플릿
- 데이터 없음: `아직 열린 제안이 없어요.` / CTA `첫 제안 작성`
- 검색 결과 없음: `검색어와 일치하는 제안이 없어요.` / CTA `추천 키워드 보기`
- 필터 과다: `조건이 너무 좁아요.` / CTA `필터 초기화`

### 8-3. 추가 규칙
- 제안 템플릿 2~3개 제공(빠른 시작)
- 검색/필터 빈 상태에 추천 필터 노출
- 관리자 빈 상태에는 운영 액션 노출(숨김글 보기 등)

### 8-4. 지표
- 빈 상태 CTA 클릭률
- 빈 상태 이탈률
